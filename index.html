Jenkins新版本windows踩坑记录

发表于 2021-01-08 分类于 运维

Jenkins Windows下使用（2.263.3以上版本）问题

此次因为项目重新部署了服务器，所以持续集成、项目数据库、项目代码都需要迁移。
但是发现在迁移过程中，新版本的Jenkins在windows下运行问题多多（不知道官方是怎么发布出来的）
我从jenkins 2.263.3版本换到2.274版本全都试过，不是windows安装后插件无法更新就是根据网上的配置修改了UpdateCenter的Https地址后
Git插件无法下载，插件即便是下载下来由于我用的Gitee，代码也无法克隆等一系列问题。

如何解决新版本一系列问题

当我发现这些问题后，在网上找资料，修改过UpdateCenter、修改过插件default.json的地址配置，一切都是徒劳，问题越用越多。
我觉得使用正常的安装模式已经是很难达到我的使用要求。所以找了另类的一套方案。
我的目的很简单，只要jenkins对应我需要的插件能在windows服务器上正常的使用
而且docker单独部署还需要跨容器访问，这块我暂时没有接触过，对于未知也是有一定的部署和学习风险。所以不使用这样的方案。
第一步，因为我旧服务器使用的是2.176版本，属于旧版本，但是能正常使用，而唯一的不同是每次登陆都会有一大堆的插件不兼容问题。
然后我将Jenkins根目录下的plus插件全部压缩打包，在官网同时又下载了最新的2.263.3的War包。
将他们放到了新的服务器中。
第二步，在新的服务器中继续使用2.176版本的安装包，安装非常快，安装完成后，我将plus插件压缩包解压放到的jenkins目录中
将war包也解压缩覆盖了Jenkins主目录的文件。
然后打开localhost:8080端口jenkins，同样查找默认密码登陆
发现登陆后可以正常搜索插件（只有部分插件无法更新），进入系统后，点击了jenkins本地更新，更新到2.263.3版本重启。
重启登陆后，更新必要的插件，发现一切都如同旧版本一样在2.263.3版本下持续集成毫无问题。

此新版jenkins在windows上部署安装，git获取代码发布困扰了我整整3天，此方案也是突发奇想发现的，所以特意记下来，希望将来jenkins在新的后续版本中能够使用更加的方便。

ABP框架结构解释

  1. Domain.shared

    此层是放模块共享的东西，如公用的枚举、常量

  2. Domain

    此层是放实体、聚合根、领域服务、值类型、仓储接口

  3. Application.Contracts

    此层放应用服务接口、和DTOs

  4. Application

    此层是放应用服务实现

  5. EntityFrameWork.Core

    此层是EF Core实现

  6. HttpApi

    此层是模块WebApi实现

  7. HttpApi.Client

    此层是对应第三方API接口

  8. Web

    此层是对应MVC或者其他前端

第一次群晖NAS使用和服务器搭建

发表于 2020-11-17 更新于 2020-11-26 分类于 硬件运维

为什么选择群晖？

也是一些机缘巧合之下，听到了群晖NAS这个产品。自己也是做了多年开发，而且云产品的服务器也将要到期。不如试试群晖来搭建自己需要的服务器？
回归正题为啥看上了群晖？

  1. 作为多年的研发玩家，服务器还是比较必须的。而有自己的机器更加自由。
  2. 现在的群晖DSM系统已经比较完善（没有了解过黑群晖，比较懒不想折腾），带 + 的型号全部都支持docker，这让群晖不仅仅只是文件服务器、云盘、网站，可玩性特别高。
  3. 提供了quickconnection方便任何地方管理你的NAS，说白了还是看上了正版群晖的服务。当然黑群晖也是非常不错的。

购买

自己的资金和日常需求也并不是要求很高，所以综合之下最后选了220+，算是比较亲民的在某东双11还能减免400+。所以其他渠道就没考虑了。

硬盘和内存升级

硬盘不用多说，因为是组raid阵列，最好是双硬盘同样大小，系统基本上都能支持。
内存是参考了许久，因为官方给出的220+是最大内存只支持6g，感觉过于的小。但是论坛贴吧等都说内存不止，甚至有单加16g的大佬。
而且基本上某东上买三星的内存都能支持。所以我是加了三星8g内存，能够正常识别，也没有使用上的问题。感觉不错。

内网服务器搭建，docker使用

既然是选择了群晖带 + 型号，当然更多的是看上了Docker方面。当我拿到手时也是毫不犹豫的选择安装了Docker套件，没有任何意外所有渠道的容器镜像是都能支持的，而且自带了Docker hub源等，特别方便。
至于容器的端口映射和如何访问，Docker命令特别多，资料丰富。就不详细说了。

外网搭建条件，和一些内网穿透办法

对于搭建服务器，更多的当然是希望能够任何地方都能够使用到。原本的群晖NAS就是能够达到这样的目的，但是也是有一定的条件的。

  1. 最方便的就是拥有一个公网IP

  2. 内网穿透产品

  1. 公网服务器做端口反向代理
    使用公网VPS+FRP内网穿透工具，实现群晖NAS服务器公网访问。
    在拥有公网地址的VPS服务器上安装FRPS，修改好对应的frps.ini服务配置信息，开始后台静默启动。（这部分有一键服务端脚本一键脚本）
    而frps配置和其原理使用参考gofrp官网
    里面既有frps配置的详细解释，还有frpc（frp client）配置方式。
    剩下的就是在群晖NAS中开启frpc连通frps实现端口代理公网访问。

  1. 帐套设置-》支付设置-〉C扫B码、B扫C码，还有不同的支付方式跟底下的设置有什么区别
    在银联和易生支付，同时这两方支持微信扫码支付和支付宝扫码支付（非原生微信支付和支付宝支付）
  2. 分销管理的流程
    多级分销，设置分销商，分销策略，佣金金额，同时接入到小程序中
  3. 小程序商品可见模版和不可见模版
    用于针对指定客户的小程序商品可见或不可见
  4. 待办提醒从哪来?
    有task任务，通过执行脚本和权限提供跳转链接
  5. 电子秤接口功能的作用
    用于pos端打条码秤重
  6. 销售单和销售订单的区别?
    销售订单相当于预销售，销售预购，不进行财务记账，不操作库存，销售单是实际销售单据

    相当于商品组装，用于生产，系统有生产模块
  9. 商品盘点是如何发起?
    盘点是随时发起，可以根据商品盘点，不锁定库存，直接盘盈盘亏
  10. 扣率代销是个什么意思?
    软件使用方代第三方销售
  11. 什么是反红冲?
    红冲是逆流程操作，反红冲是红冲的逆流程操作

leetcode_1471

发表于 2020-06-20 更新于 2020-11-05 分类于 算法

题目

给你一个整数数组 arr 和一个整数 k 。
设 m 为数组的中位数，只要满足下述两个前提之一，就可以判定 arr[i] 的值比 arr[j] 的值更强：
|arr[i] - m| > |arr[j] - m|
|arr[i] - m| == |arr[j] - m|，且 arr[i] > arr[j]
请返回由数组中最强的 k 个值组成的列表。答案可以以 任意顺序 返回。
中位数 是一个有序整数列表中处于中间位置的值。形式上，如果列表的长度为 n ，那么中位数就是该有序列表（下标从 0 开始）中位于 ((n - 1) / 2) 的元素。
例如 arr = [6, -3, 7, 2, 11]，n = 5：数组排序后得到 arr = [-3, 2, 6, 7, 11] ，数组的中间位置为 m = ((5 - 1) / 2) = 2 ，中位数 arr[m] 的值为 6 。
例如 arr = [-7, 22, 17, 3]，n = 4：数组排序后得到 arr = [-7, 3, 17, 22] ，数组的中间位置为 m = ((4 - 1) / 2) = 1 ，中位数 arr[m] 的值为 3 。

示例 1：
输入：arr = [1,2,3,4,5], k = 2
输出：[5,1]
解释：中位数为 3，按从强到弱顺序排序后，数组变为 [5,1,4,2,3]。最强的两个元素是 [5, 1]。[1, 5] 也是正确答案。
注意，尽管 |5 - 3| == |1 - 3| ，但是 5 比 1 更强，因为 5 > 1 。

示例 2：
输入：arr = [1,1,3,5,5], k = 2
输出：[5,5]
解释：中位数为 3, 按从强到弱顺序排序后，数组变为 [5,5,1,1,3]。最强的两个元素是 [5, 5]。

示例 3：
输入：arr = [6,7,11,7,6,8], k = 5
输出：[11,8,6,6,7]
解释：中位数为 7, 按从强到弱顺序排序后，数组变为 [11,8,6,6,7,7]。
[11,8,6,6,7] 的任何排列都是正确答案。

示例 4：
输入：arr = [6,-3,7,2,11], k = 3
输出：[-3,11,2]

示例 5：
输入：arr = [-7,22,17,3], k = 2
输出：[22,17]

提示：
1 <= arr.length <= 10^5
-10^5 <= arr[i] <= 10^5
1 <= k <= arr.length

解题思路

常规思路

  1. 先算出当前数组中位数 arr[(arr.length-1)/2]=?
  2. 再循环比较，形成新的数组，然后取前k个值

其他思路

  1. 发现每一位减去中位数中，越靠近中位数的绝对值越小（也就是题目的越弱），越靠近两端的值越大（也就是越强），是否可以将数组拆分成小于中位数数组和大于中位数数组，然后进行绝对值计算形成一个数组，可以减少比较次数？
